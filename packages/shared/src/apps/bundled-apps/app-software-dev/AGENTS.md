# 软件开发工作区

你正在一个**软件开发工作区**中工作。这是一个专为开发者设计的 AI 助手环境，支持代码开发、测试、文档和工具构建等软件开发工作流。

## 工作区特点

- **开发辅助**：代码审查、重构建议、调试协助
- **自动化**：变更日志生成、测试生成、文档生成
- **工具开发**：MCP 服务器开发指南

## 目录结构

```
src/       # 源代码
tests/     # 测试文件
docs/      # 文档
scripts/   # 脚本工具
```

## 内置技能

### 1. 变更日志生成 (changelog-generator)
根据 Git 提交历史自动生成用户友好的变更日志。

**功能**：
- 扫描 Git 提交历史
- 智能分类变更（功能、改进、修复、破坏性变更）
- 将技术提交转换为用户可读的描述
- 过滤内部提交（重构、测试等）

**使用场景**：
- 准备新版本发布说明
- 创建周报/月报更新摘要
- 生成应用商店更新描述
- 维护公开的产品更新页面

**示例用法**：
```
创建从上次发布以来的变更日志
生成过去一周所有提交的变更日志
为 v2.5.0 版本创建发布说明
```

### 2. MCP 服务器开发 (mcp-builder)
创建高质量的 MCP (Model Context Protocol) 服务器，让 LLM 与外部服务交互。

**功能**：
- MCP 服务器架构设计
- Python (FastMCP) 和 TypeScript (MCP SDK) 支持
- API 集成最佳实践
- 工具设计模式

**开发流程**：
1. 深入研究和规划
2. 实现核心功能
3. 审查和优化
4. 测试和部署

## 工作原则

### 代码质量
- **可读性优先**：代码应该易于理解和维护
- **测试覆盖**：重要功能需要有测试
- **文档完善**：关键函数需要文档说明

### Git 工作流
- **小步提交**：每次提交做一件事
- **有意义的信息**：提交信息要清晰描述变更
- **版本控制**：使用语义化版本号

### 错误处理
- **优雅失败**：预期可能的错误并处理
- **清晰反馈**：错误信息要能指导修复
- **日志记录**：关键操作要有日志

## 常见工作场景

### 发布准备
1. 审查自上次发布以来的提交
2. 生成变更日志
3. 更新版本号
4. 创建发布标签

### 代码审查
1. 分析代码变更
2. 检查代码质量
3. 提供改进建议
4. 确认测试覆盖

### 新功能开发
1. 理解需求
2. 设计实现方案
3. 编写代码和测试
4. 文档和提交

## 提示

- 运行命令前确保在正确的 Git 仓库目录
- 生成的变更日志需要人工审核后再发布
- MCP 服务器开发建议先做原型验证
- 遵循项目已有的代码风格和规范
