# 工作区与会话使用指南

本文档介绍 CreatorFlow 中工作区（Workspace）和会话工作目录（Working Directory）的概念及使用场景。

## 核心概念

### 工作区（Workspace）

工作区是 CreatorFlow 的顶级组织单元，用于分隔不同的项目、客户或工作上下文。

**特点：**
- 每个工作区完全独立，拥有自己的会话、数据源、技能和配置
- 工作区之间的数据互不干扰
- 可以为不同客户、项目或生活领域创建独立工作区
- 支持多窗口，每个窗口可以打开不同的工作区

**包含内容：**
- 会话历史记录
- 数据源配置（API、MCP 服务、本地文件夹）
- 技能（Skills）
- 标签（Labels）
- 状态配置（Todo States）
- 权限设置

### 会话工作目录（Working Directory）

会话工作目录是单个会话的执行上下文，决定 AI 助手在执行文件操作和命令时的基准目录。

**特点：**
- 每个会话可以设置独立的工作目录
- 同一工作区内的不同会话可以有不同的工作目录
- 可以在工作区设置中配置默认工作目录
- 会话创建时自动继承工作区的默认工作目录

## 使用场景

### 场景一：软件开发团队

**工作区设置：**
```
工作区: 电商平台项目
├── 会话1: 前端开发 (工作目录: /projects/ecommerce/frontend)
├── 会话2: 后端 API (工作目录: /projects/ecommerce/backend)
├── 会话3: 移动端 App (工作目录: /projects/ecommerce/mobile)
└── 会话4: 部署运维 (工作目录: /projects/ecommerce/devops)
```

**优势：**
- 并行处理前端、后端、移动端的开发任务
- 每个会话专注于特定模块，上下文清晰
- 代码操作不会跨目录混淆

## 最佳实践

### 1. 工作区命名
- 使用清晰、具体的名称
- 避免过于笼统的命名（如"工作"、"项目"）
- 推荐格式：`客户名-项目名` 或 `领域-子领域`

### 2. 工作目录设置
- 在工作区设置中配置默认工作目录
- 新会话自动继承默认目录，减少重复设置
- 需要时可在会话中临时切换目录

### 3. 会话管理
- 使用标签（Labels）对会话分类
- 使用状态（States）追踪任务进度
- 定期归档已完成的会话

### 4. 数据源配置
- 为工作区配置相关的数据源
- 不同工作区可以连接不同的 API 和服务
- 敏感数据源只在需要的工作区启用

## 常见问题

### Q: 工作区之间可以共享会话吗？
A: 不可以。会话属于特定工作区，无法跨工作区移动。如需复用内容，可以导出会话或复制关键信息。

### Q: 一个会话可以同时操作多个目录吗？
A: 可以。工作目录是默认基准目录，但在对话中可以指定操作其他路径的文件。

### Q: 如何快速切换工作区？
A: 
- 使用左下角的工作区选择器
- 使用快捷键（可在设置中自定义）
- 在新窗口中打开另一个工作区

### Q: 删除工作区会丢失所有数据吗？
A: 是的。删除工作区会永久删除其中的所有会话、配置和数据。请谨慎操作。

## 总结

| 特性 | 工作区 (Workspace) | 会话工作目录 (Working Directory) |
|------|-------------------|--------------------------------|
| 作用域 | 全局组织单元 | 单个会话 |
| 包含内容 | 会话、数据源、配置等 | 文件操作的基准路径 |
| 数据隔离 | 完全隔离 | 同一工作区内共享配置 |
| 典型用途 | 分隔项目/客户/领域 | 指定代码/文件操作位置 |
| 可修改性 | 可随时切换 | 可在会话中随时更改 |

合理使用工作区和会话工作目录，可以让你的 AI 助手工作流程更加有序、高效。
