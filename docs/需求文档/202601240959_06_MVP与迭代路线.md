## MVP（最小可行产品）定义

### 核心目标
验证"AI Agent 驱动的脚本创作 + 智能素材管理"的核心价值，快速获取用户反馈。

### MVP 范围

#### ✅ 必须包含的功能

**1. 基础 Agent 对话系统**
- 自然语言交互创建项目
- 理解创作意图并生成脚本大纲
- 多轮对话优化脚本内容
- 基于 CreatorFlow 的会话管理
- 工具调用可视化

**2. 脚本创作核心功能**
- 结构化脚本编辑器（基于 Monaco Editor）
- AI 辅助脚本生成（基于 Claude API）
- 脚本版本管理（最多保留 10 个版本）
- 脚本段落标注（素材需求、配音、字幕）
- Markdown 支持

**3. 基础素材管理**
- 本地素材扫描和导入
- 简单的文件夹分类
- 基础的标签系统（手动打标签）
- 素材与脚本的手动关联
- 素材预览（图片、视频）

**4. 图文内容生成**
- 脚本 → Markdown 转换
- 基础排版模板（3-5 个模板）
- 图片插入和排版
- 导出为图片或 PDF
- 小红书、公众号风格适配

**5. 项目管理**
- 项目列表和状态管理
- 项目与脚本、素材的关联
- 基础的搜索和筛选
- 项目归档

**6. 本地存储**
- 所有数据本地存储（JSONL + SQLite）
- 加密存储敏感信息
- 自动保存

#### ❌ MVP 不包含（后续迭代）

- ❌ 视频自动生成（技术复杂度高）
- ❌ 全网素材聚合（需要对接多个 API）
- ❌ AI 素材智能推荐（需要训练推荐模型）
- ❌ 深度学习和个性化（需要积累用户数据）
- ❌ 跨平台发布（需要对接各平台 API）
- ❌ 云端同步（MVP 先纯本地）
- ❌ 团队协作功能
- ❌ 移动端支持

## 三阶段迭代路线图

### 阶段 1：MVP（3-4 个月）

**目标**：验证核心价值，获取早期用户反馈

**时间线**：
- 第 1 个月：技术验证和基础架构
- 第 2-3 个月：核心功能开发
- 第 4 个月：封闭测试和优化

**功能清单**：
- ✅ Agent 对话系统
- ✅ 脚本创作工作台
- ✅ 基础素材管理
- ✅ 图文内容生成
- ✅ 项目管理
- ✅ 本地存储

**技术栈**：
- 桌面端：Electron + React + TypeScript
- AI：Claude API
- 存储：SQLite + JSONL
- UI：Tailwind CSS + shadcn/ui

**用户目标**：
- 种子用户：10-50 人
- 目标用户：图文创作者（小红书、公众号）
- 反馈渠道：用户访谈、问卷调查

**成功指标**：
- 用户留存率 > 30%（7 日）
- 项目完成率 > 50%
- 用户满意度 > 4.0/5.0
- 平均创作时长 < 30 分钟

## 技术债务管理

### 已知技术债务

**1. 视频生成性能**
- 问题：本地渲染可能很慢
- 影响：用户体验
- 解决方案：阶段 2 引入云端渲染
- 优先级：中

**2. AI 成本控制**
- 问题：Claude API 调用成本高
- 影响：运营成本
- 解决方案：优化提示词、使用缓存、引入本地模型
- 优先级：高

**3. 大文件存储**
- 问题：视频素材占用大量存储
- 影响：用户体验、成本
- 解决方案：云端存储 + CDN
- 优先级：中

**4. 数据同步冲突**
- 问题：多设备同步可能冲突
- 影响：数据一致性
- 解决方案：CRDT 或 OT 算法
- 优先级：低（阶段 2 再解决）

### 技术风险缓解策略

**1. 降低视频生成风险**
- MVP 先做图文，验证核心流程
- 视频生成分阶段实现（先简单拼接，再高级编辑）
- 提供"半自动"模式，降低用户期望

**2. 控制 AI 成本**
- 设计合理的用量限制
- 使用 Haiku 处理简单任务
- 缓存常见请求结果
- 批量处理降低调用次数

**3. 优化存储成本**
- 本地优先，云端可选
- 压缩和去重
- 分层存储（热数据 SSD，冷数据 HDD）
