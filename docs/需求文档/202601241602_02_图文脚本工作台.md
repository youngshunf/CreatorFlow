# 图文脚本工作台 PRD（小红书 / 抖音 / 公众号）

> 版本：v0.1 草案  
> 关联文档：00_创流CreatorFlow_产品总览、01_创作者选题与规划工作流

---

## 1. 背景与目标

### 1.1 背景

选题确定之后，个人创作者需要在不同平台上写出风格各异的内容：

- 小红书图文：更生活化、故事感、种草导向；
- 抖音短视频：强钩子 + 高密度信息 + 口播友好；
- 公众号长文：结构清晰、论述充分、适合深度阅读。

目前创作者通常在多个工具之间来回切换（备忘录、文档、小红书/公众号编辑器），缺乏一个统一的「脚本/文案中枢」。

### 1.2 目标

- 为每个选题提供一个统一的「脚本/文案工作台」，支撑多平台派生；
- 以图文为核心，兼顾短视频脚本，减少重复输入；
- 支持 AI 多版本生成与对比、改写/续写等高频操作；
- 保留结构化信息，为后续排版、发布、数据分析打好基础。

---

## 2. 对象模型与概念

### 2.1 Script（脚本）

- 每个选题对应 1 个或多个 Script，对应不同平台和形态；
- Script 应包含：
  - 基本信息：
    - 关联项目、选题、目标平台（可多选）、内容形态（图文/短视频/长文）；
  - 结构化内容：
    - 段落/章节列表；
    - 对于短视频：镜头/时长/口播文本；
  - 元信息：
    - 创建人、创建时间、最近修改时间；
    - AI 参与程度（AI 初稿/人工主导/混合）。

### 2.2 平台视图

- 对于一个选题，可存在多个平台视图：
  - `ScriptView.XHS`：小红书图文版本；
  - `ScriptView.DY`：抖音短视频版本；
  - `ScriptView.WX`：公众号长文版本；
- 同一脚本在内部共享核心信息，但允许对每个平台做差异化调整。

---

## 3. 用户流程

### 3.1 从选题进入脚本工作台

1. 用户在「选题规划」中勾选若干选题并点击「创建脚本」；
2. 系统为每条选题创建对应的 Script 草稿：
   - 根据推荐平台自动生成初始平台视图（如只推荐小红书，则只生成 XHS 视图）；
3. 用户点击某一条 Script 进入「脚本工作台」。

### 3.2 在脚本工作台中创作

1. 选择平台视图（顶部 tab：小红书 / 抖音 / 公众号）；
2. 在编辑区：
   - 查看/编辑 AI 生成的初稿；
   - 使用右侧工具栏进行改写/扩写/缩写、多版本生成；
3. 可切换视图查看不同平台版本差异；
4. 创作完成后，将脚本状态标记为「已完成，待排版」。

## 4. 功能需求

### 4.1 初稿生成（P0）

- 对于从选题生成的脚本草稿，系统应支持：

#### 4.1.1 小红书图文脚本

- 结构：
  - 开头钩子（1 段）；
  - 故事/场景（1–3 段）；
  - 干货/要点（3–5 条）；
  - 总结 + 行动号召（1 段）；
- 要求：
  - 文风贴合小红书（口语化、Emoji 可选）；
  - 每段长度适中（方便后续排版）。

#### 4.1.2 抖音短视频脚本

- 结构：
  - 开头 3–5 秒钩子（1–2 句，对应画面描述）；
  - 主体 20–40 秒内容（分 3–5 个信息点）；
  - 结尾 3–5 秒 CTA；
- 对脚本结构化存储：
  - 镜头编号、预计时长、口播文本、画面建议。

#### 4.1.3 公众号长文脚本

- 结构：
  - 引言（问题背景/痛点）；
  - 主体章节（2–4 个一级标题，每个下有若干要点）；
  - 小结与行动号召；
- 要求：
  - 逻辑清晰、有小标题；
  - 为后续排版模块提供结构标记（H1/H2/H3）。

### 4.2 AI 写作增强功能（P0）

在脚本编辑器右侧或悬浮工具栏提供：

- 改写：
  - 保留大意，重写语句；
- 扩写：
  - 对选中段落进行详细展开；
- 缩写：
  - 保留关键信息，缩短长度；
- 续写：
  - 在光标处继续写下去；
- 多版本生成：
  - 一次生成 2–3 个不同风格版本，用户可对比选择；
- 平台风格切换：
  - 将一段公众号风格文字，转换为小红书风格或抖音口播风格。

### 4.3 版本管理（P1）

- 每个 Script 保留版本历史：
  - 自动保存关键修改；
  - 支持标记「重要版本」；
  - 提供简单的版本对比（文本 diff）。

### 4.4 脚本与素材/结构标注（P1）

- 支持在脚本中为某些段落/镜头添加标注：
  - 需要的素材类型（图片/视频/截图等）；
  - 预期画面风格；
- 这些标注将提供给「多平台排版与发布模块」和未来的素材系统使用。

---

## 5. 交互与体验要求

- 脚本编辑器应具备：
  - 基础富文本能力（加粗、列表、引用等）；
  - 清晰的段落/章节结构展示；
  - 快捷键支持（如常见的 Cmd+B/Cmd+I 等）。
- AI 操作需提供：
  - 流式显示结果，避免长时间空白等待；
  - 对原文与新版本进行并排展示，方便对比。

---

## 6. 与其他模块的关系

- 与「选题与规划」模块：
  - 从选题生成脚本草稿；
  - 将脚本状态反馈回选题列表（如标记哪些选题已完成脚本）。
- 与「多平台排版与发布」模块：
  - 为排版模块提供结构化文本数据；
  - 为发布模块提供平台对应的最终文案与脚本。
- 与「作品 & 数据看板」模块：
  - 通过脚本与作品的关联，实现「从数据回看脚本」的能力（长远目标）。

---

## 7. 验收标准（v1）

- 从一个选题开始，用户可以在 1 分钟内得到：
  - 一个小红书图文初稿；
  - 一个抖音短视频脚本结构；
  - 一个公众号长文大纲和初稿；
- 用户在同一界面能对上述三种视图进行：
  - 基础编辑；
  - 常用 AI 操作（改写/扩写/续写/缩写/多版本）；
- 生成的内容「风格基本符合平台特点」，不会出现明显的平台不匹配（例如抖音脚本中出现大量公众号排版语气）。
