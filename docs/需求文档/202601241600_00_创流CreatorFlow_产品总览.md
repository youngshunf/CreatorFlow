# 创流 CreatorFlow - 产品定义文档（个人创作者版）

> 版本：v0.9（草案）  
> 适用范围：以**个人自媒体创作者**为主，平台优先支持 **小红书 + 抖音 + 公众号**，以**图文为核心，图文转短视频为扩展**。

---

## 1. 产品概述

### 1.1 产品名称

- 中文名称：**创流**  
- 英文名称：**CreatorFlow**

### 1.2 核心定位

创流是一个面向**个人自媒体创作者**的 **AI 创作操作系统**：

- 以 **图文脚本/文案** 为核心资产；
- 通过 **AI Agent 对话** 驱动，从「今天发什么」到「怎么写/怎么讲/怎么发」；
- 打通 **选题 → 创作 → 排版 → 发布 → 运营** 的完整闭环；
- 先深度支持 **小红书 / 抖音 / 公众号** 三大平台，再向更多平台和团队/企业场景扩展。

### 1.3 目标用户

**主要目标用户（MVP 聚焦）**：

- 日更 / 周更的个人创作者：
  - 常驻平台：小红书 / 抖音 / 公众号；
  - 关注点：持续输出、内容质量、稳定节奏；
  - 痛点：
    - 每天「今天发什么」的选题焦虑；
    - 写文案、排版、准备多平台版本的耗时；
    - 发布 & 数据查看分散在多个平台后台；

**次要用户（后续扩展）**：

- 小型创作工作室 / MCN（3–10 人小团队）；
- 品牌新媒体团队（有公众号 + 抖音号等矩阵账号）；
- 内容为主的知识工作者、科技工作者（后续再细化）。

### 1.4 用户核心问题

1. **今天发什么？**
   - 如何快速找到既符合自己定位、又有数据潜力的选题？

2. **怎么写 / 怎么讲？**
   - 如何写出适配不同平台风格的图文/脚本？
   - 如何把同一主题拆成图文 + 短视频多种形式？

3. **怎么发到多个平台？**
   - 小红书 / 抖音 / 公众号 的发布规范和节奏不同，如何一处创作，多处发布？

4. **下次怎么做得更好？**
   - 各平台数据分散，如何看整体表现？
   - 如何基于过往表现，优化后续选题和内容？

## 2. 一句话价值主张

> **用创流，一个工作台搞定自媒体创作全流程：AI 帮你选题、写稿、排版、发小红书/抖音/公众号，并根据数据给出下一次该怎么更好的建议。**

---

## 3. 关键价值点（对比当前创作者常见工作方式）

### 3.1 全流程一体化，而非工具拼凑

- 现在：
  - 选题→备忘录/脑图；
  - 写稿→Notion/飞书文档；
  - 排版→小红书/公众号编辑器；
  - 数据→各平台后台自己看、自己总结；
- 创流：
  - 从对话开始，统一在一个「作品/脚本」对象上完成：选题、脚本、排版、发布、数据回流与分析。

### 3.2 Agent 驱动的「数字分身」

- 不只是「文案生成工具」，而是：
  - 能理解「这周内容策略」的 AI 助手；
  - 能记住创作者个人风格和偏好的长期搭档；
  - 能根据数据主动建议「下周怎么发」的运营顾问。

### 3.3 图文为核心的多形态内容工厂

- 统一用「脚本/文案」表达内容逻辑：
  - 小红书图文；
  - 公众号长文；
  - 抖音短视频口播脚本与分镜；
- 一次创作，多处使用，减少重复劳动。

### 3.4 面向三大平台的深度适配

- 深度支持：小红书 + 抖音 + 公众号；
- 在选题、创作、排版、发布等每个环节，都会考虑平台的：
  - 风格 & 文案习惯；
  - 内容结构与节奏偏好；
  - 封面、标题、标签等规则限制。

### 3.5 数据闭环驱动的持续优化

- 每条作品是一个长期资产：
  - 关联原始脚本、实际发布版本、多平台数据；
- AI 能够：
  - 总结哪些选题 / 结构 / 发布时间表现最佳；
  - 给出下一阶段的选题规划和内容优化建议。

## 4. 首版产品边界（v1 / MVP）

> 目标：在 3–4 个月内交付一个**可以被真实个人创作者持续使用**的端到端闭环。

### 4.1 必须具备的能力（P0）

1. **选题 & 规划（简化版）**
   - 支持针对小红书/抖音/公众号的选题推荐；
   - 能基于项目/账号定位，生成 3–7 天的发文规划；
   - 每个选题带有简要结构和平台建议。

2. **脚本/文案工作台（图文优先）**
   - 针对单个选题，生成：
     - 小红书图文结构与文案初稿；
     - 抖音短视频口播稿（30–60 秒，带简单分镜说明）；
     - 公众号长文结构与文案初稿；
   - 支持多版本生成与对比；
   - 支持简单的「偏好学习」（如开头风格、用词偏好）。

3. **平台化排版能力**
   - 提供小红书/公众号 的排版模板与自动排版；
   - 能一键预览不同平台的展示效果；
   - 抖音方向先聚焦在「脚本/字幕文本」整理，而不是完整剪辑。

4. **多平台发布（首批：小红书 + 抖音 + 公众号）**
   - 账号绑定与安全存储；
   - 从单个作品中选择平台，一次设置标题封面文案，支持：
     - 立即发布；
     - 定时发布；
   - 至少保证小红书/公众号的体验相对顺滑，即便某些平台阶段性需要半自动（复制内容+跳转平台后台）。

5. **作品 & 数据回流**
   - 统一的「作品列表」视图：每条作品聚合三平台发布记录；
   - 定时拉取基础数据：浏览/阅读、点赞、评论、收藏等；
   - 简单的 AI 运营点评：
     - 单条作品「为什么表现好/一般」；
     - 最近一段时间的「话题/结构/发布时间」总结与建议。

### 4.2 延后到 v2+ 的能力（P1+）

- 完整的视频自动生成与剪辑；
- 深度团队协作与企业账号/权限体系；
- 更大范围的平台支持（B 站、微博等）；
- 多模型选择界面、复杂的 Agent 工作流可视化编辑；
- 完整素材三层体系与大型知识库。
